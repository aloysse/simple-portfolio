## 專案概述

住通房管是住通科技股份有限公司面向房仲業者所開發的 SaaS 管理系統，以多分店、多人員的仲介公司為核心使用情境。系統涵蓋人員與分店管理、行政作業自動化、委託案件與成交物件的全流程追蹤，以及客戶關係維護（CRM）等功能模組，協助仲介業者提升日常作業效率並集中管理業務資料。

系統採用 React + TypeScript + MUI 技術棧，以 SaaS 多租戶架構運作，支援不同規模的仲介公司共用同一平台，並透過租戶隔離確保資料安全。

## 👨‍💻 我的角色

**Front-End Developer / 前端架構設計**

- 深入了解業務端需求，將業務流程轉化為具體的前端功能規格

- 安排前端團隊的工作分配與開發優先順序，確保專案進度
- 協作開發核心功能模組，並建立可重用元件庫與前端開發規範
- 參與 UI/UX 規劃，協助優化管理介面的操作流暢度

---

## ❓ 挑戰

1. **複雜業務流程的前端化**：仲介業的作業流程涉及多個角色（業務員、主管、後台行政）與多種物件狀態（委託中、洽談中、成交），如何在前端清晰呈現並管理這些複雜狀態是首要難題。
2. **多分店多人員的權限架構**：不同分店、不同職級的人員需要差異化的介面與功能權限，前端需要靈活的角色控制機制。
3. **跨功能模組的一致性**：系統涵蓋人員、物件、客戶、報表等多個模組，需在統一的設計語言下保持互動一致性，避免各模組風格分歧。
4. **前端團隊協作規範**：在多人開發環境下，需要建立清晰的元件設計規範與程式碼標準，確保協作品質。

---

## 🎯 目標

1. 將業務需求準確轉化為前端技術規格，降低開發過程中的需求落差。
2. 建立可擴充的前端架構，支援後續新功能的快速迭代。
3. 統一前端開發規範，提升團隊協作效率與程式碼品質。
4. 打造直覺、易用的管理介面，降低仲介業者的學習成本。

---

## ✨ 解決方案

- **需求分析與架構規劃**：與業務端深度溝通，梳理完整的作業流程，設計符合業務邏輯的前端資料流與元件結構。
- **元件化設計**：以 MUI 為基礎封裝業務導向的可重用元件，統一管理 Props 介面與 TypeScript 型別定義，減少重複開發。
- **權限控制機制**：設計靈活的前端角色權限架構，支援依分店、職級動態控制頁面可見性與操作權限。
- **開發規範建立**：制定元件設計準則、命名慣例與 code review 標準，提升跨成員的協作效率，縮短新成員的熟悉時間。
