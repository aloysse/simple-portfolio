## 專案概述

住通房管系統是住通科技股份有限公司旗下的 SaaS 租賃管理平台，專為房東、租戶與房屋管理業者設計，提供物件管理、租約追蹤、費用收付與維修申報等一站式服務。平台橫跨 Web 後台管理系統與 iOS / Android 行動應用程式，協助使用者隨時掌握資產狀況，大幅降低房屋管理的人工成本與溝通摩擦。

系統以 SaaS 模型運作，同時服務多個租戶（Tenant），因此前端架構需具備高度彈性與可擴充性。Web 端採用 React + TypeScript 搭配 Ant Design 元件庫，行動端則以 React Native 開發，讓雙端共用商業邏輯與部分 UI 元件，確保功能迭代效率。

我於 2025 年 2 月加入住通科技，負責前端開發與架構設計，並協助規劃前端部門的開發流程與標準，確保跨平台產品在快速迭代的前提下維持一致的使用體驗與程式碼品質。

## ❓ 挑戰

1. **跨平台一致性**：Web 與 App 雙端需呈現相同的業務邏輯與互動體驗，但兩者底層渲染機制差異大，如何在不重複開發的前提下維持一致性是一大難題。
2. **多租戶架構的前端複雜度**：SaaS 系統需支援不同規模的用戶，各租戶可能有不同的功能開關與 UI 需求，前端需具備足夠的彈性來應對動態配置。
3. **App 上架與環境管理**：iOS 與 Android 的建置流程、簽署憑證、環境變數分離及版本管理各有差異，需要一套標準化流程才能穩定交付。
4. **開發規範與協作效率**：前端團隊在快速迭代的壓力下，元件設計缺乏統一標準，導致重複實作與維護困難，亟需建立可共用的元件體系與開發準則。

---

## 🎯 目標

1. 建立前端可重用元件庫，提升 Web 與 App 雙端的開發效率與視覺一致性。
2. 規劃標準化的 App 建置與上架流程，確保 iOS / Android 版本能穩定且快速地交付。
3. 制定前端架構設計原則與開發規範，降低技術債並提升團隊協作品質。
4. 參與 UI/UX 規劃，確保管理系統與行動 App 的使用者體驗流暢直覺，符合不同使用情境的需求。

---

## ✨ 解決方案

- **元件化架構設計**：以 Ant Design 為基礎，封裝符合業務需求的可重用元件，並整合 TypeScript 型別定義，減少跨頁面的重複實作，提升整體開發效率。
- **React Native 跨平台開發**：採用 React Native 開發雙平台 App，共用核心業務邏輯與 API 串接層，並針對 iOS 與 Android 的 UX 差異進行平台特定調整，確保原生體驗。
- **標準化 App 上架流程**：建立完整的建置、測試與上架 SOP，涵蓋憑證管理、環境設定（開發 / 測試 / 正式）與版本號控管，大幅縮短每次上架所需的準備時間。
- **前端開發規範建立**：制定元件設計準則、資料夾結構慣例與 code review 標準，協助前端團隊在快速迭代中保持程式碼品質，並降低新成員的學習曲線。
